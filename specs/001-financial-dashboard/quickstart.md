# 금융 분석 대시보드 — 퀵스타트 & 통합 시나리오

**작성일**: 2026-04-29

---

## 개요

이 문서는 구현 후 개발자가 주요 시나리오를 빠르게 검증할 수 있는 통합 체크리스트를 제공한다. 외부 API 없이 목업 데이터로 동작하므로 로컬 환경 세팅 후 즉시 확인 가능하다.

---

## 로컬 실행

```bash
# 의존성 설치
npm install

# 개발 서버 실행
npm run dev

# 브라우저에서 확인
open http://localhost:3000
```

---

## 핵심 시나리오 검증

### 시나리오 1: 대시보드 초기 로드

**경로**: `/` (메인 대시보드)

| 단계 | 검증 항목 |
|------|-----------|
| 1 | KPI 카드 4개가 상단에 표시된다 |
| 2 | 각 KPI 카드에 지표명, 현재값, 변화량 배지가 표시된다 |
| 3 | 증가 지표는 초록색 배지(▲), 감소 지표는 빨간색 배지(▼)로 표시된다 |
| 4 | 차트 영역이 기본 설정(1개월, 선택된 지표, 라인 차트)으로 렌더링된다 |
| 5 | 페이지 로드 시 레이아웃이 깨지지 않는다 |

---

### 시나리오 2: 차트 필터 상호작용

**경로**: `/` (메인 대시보드) — 차트 영역

| 단계 | 검증 항목 |
|------|-----------|
| 1 | 기간 필터(1주/1개월/3개월/1년)를 클릭하면 차트 데이터가 해당 기간으로 변경된다 |
| 2 | 지표 선택 드롭다운에서 다른 지표를 선택하면 차트가 즉시 업데이트된다 |
| 3 | 차트 유형 필터(라인/바/영역/도넛)를 전환하면 차트 렌더링이 변경된다 |
| 4 | 여러 차트가 있을 때 각 차트의 필터 상태가 독립적으로 유지된다 |
| 5 | 필터 변경 후 페이지를 새로고침하면 기본값으로 초기화된다 |

---

### 시나리오 3: 라이트/다크 모드 전환

**경로**: 전체 페이지 (헤더 ThemeToggle 버튼)

| 단계 | 검증 항목 |
|------|-----------|
| 1 | 헤더의 테마 전환 버튼을 클릭하면 전체 페이지 테마가 전환된다 |
| 2 | 다크 모드에서 차트, KPI 카드, 배경색이 적절한 어두운 색으로 변경된다 |
| 3 | 테마 전환 버튼에 `aria-pressed` 속성이 올바르게 설정된다 |
| 4 | 선택한 테마가 `localStorage`에 저장되어 새로고침 후에도 유지된다 |
| 5 | 라이트↔다크 전환 시 텍스트 가독성이 모든 컴포넌트에서 유지된다 |

---

### 시나리오 4: 반응형 레이아웃

**검증 방법**: 브라우저 개발자 도구에서 뷰포트 크기 조절

| 뷰포트 | 검증 항목 |
|--------|-----------|
| 모바일 (375px) | KPI 카드가 1열 또는 2열 그리드로 표시된다 |
| 태블릿 (768px) | KPI 카드가 2열 그리드, 차트가 전체 너비로 표시된다 |
| 데스크톱 (1280px+) | KPI 카드가 4열 그리드, 차트가 나란히 표시된다 |
| 전체 | 헤더 네비게이션이 뷰포트에 맞게 반응한다 |

---

### 시나리오 5: 관리자 페이지 — 지표 관리

**경로**: `/admin`

| 단계 | 검증 항목 |
|------|-----------|
| 1 | 전체 지표 목록이 테이블 형태로 표시된다 |
| 2 | 각 지표의 활성/비활성 토글 버튼이 표시된다 |
| 3 | 토글 클릭 시 UI 상태가 즉시 반영된다 (저장 전) |
| 4 | '저장' 버튼 클릭 시 `localStorage`에 상태가 저장된다 |
| 5 | 메인 대시보드로 돌아오면 비활성화한 지표의 KPI 카드가 숨겨진다 |
| 6 | 저장되지 않은 변경사항이 있을 때 시각적 표시(예: 버튼 강조)가 된다 |

---

### 시나리오 6: 빈 상태(Empty State)

| 상황 | 검증 항목 |
|------|-----------|
| 모든 지표 비활성화 | 대시보드에 EmptyState 컴포넌트가 표시된다 |
| 해당 기간에 데이터 없음 | 차트 영역에 EmptyState 컴포넌트가 표시된다 |
