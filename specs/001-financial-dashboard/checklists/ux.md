# UX 요구사항 명확성 체크리스트: 금융 분석 대시보드

**목적**: 대시보드 UI, 테마 전환, 차트 필터, 관리자 페이지 요구사항의 명확성·완전성·일관성 검증 (요구사항 자체의 품질을 테스트)
**작성일**: 2026-04-29
**기능**: [spec.md](../spec.md)
**초점 영역**: 대시보드 UI · 테마 전환 · 차트 필터 · 관리자 페이지

---

## 요구사항 완전성 (Requirement Completeness)

- [ ] CHK001 대시보드 메인 화면의 KPI 카드와 차트 배치에 대한 레이아웃 규칙(그리드 열 수, 정렬 방식)이 명시되어 있는가? [완전성, Gap, Spec §FR-001, FR-003]
- [ ] CHK002 반응형 브레이크포인트(375px, 768px, 1280px) 각각에서 KPI 카드와 차트의 정확한 배치 변화가 정의되어 있는가? [완전성, Spec §FR-018]
- [ ] CHK003 차트 필터(기간, 지표, 차트 유형) UI 컴포넌트 유형(드롭다운, 탭 버튼, 세그먼트 컨트롤 등)이 명시되어 있는가? [완전성, Gap, Spec §FR-005~007]
- [ ] CHK004 각 차트의 초기 로드 시 기본 필터 상태(기본 기간, 기본 지표, 기본 차트 유형)가 정의되어 있는가? [완전성, Gap, Spec §FR-005~007]
- [ ] CHK005 각 차트에서 전환 가능한 지표의 구체적인 목록(이름, 단위)이 명시되어 있는가? [완전성, Gap, Spec §FR-006]
- [ ] CHK006 테마 전환 버튼/토글의 위치, 아이콘 형태, 레이블 유무가 명시되어 있는가? [완전성, Gap, Spec §FR-009]
- [ ] CHK007 다크 모드에서 사용할 차트 색상 팔레트(구체적 색상값 또는 기준)가 명시되어 있는가? [완전성, Gap, Spec §FR-010]
- [ ] CHK008 관리자 페이지에서 대시보드로 이동할 수 있는 내비게이션 수단(링크, 버튼)이 명시되어 있는가? [완전성, Gap, Spec §FR-013]
- [ ] CHK009 관리자 페이지의 지표 목록 표에 표시해야 할 컬럼(이름, 단위, 현재 값, 상태 등)이 명시되어 있는가? [완전성, Gap, Spec §FR-014]
- [ ] CHK010 관리자 페이지의 차트 구성 표에 표시해야 할 정보(차트 제목, 유형, 연결 지표 등)의 항목이 명시되어 있는가? [완전성, Gap, Spec §FR-015]

## 요구사항 명확성 (Requirement Clarity)

- [ ] CHK011 관리자 페이지가 "일반 대시보드 UI와 시각적으로 구분된다"는 표현이 측정 가능한 기준(색상 차이, 헤더 문구, 배경 등)으로 구체화되어 있는가? [명확성, Spec §US4]
- [ ] CHK012 US3 수락 기준의 "모든 UI 요소의 색상이 즉시 전환된다"에서 '모든 UI 요소'의 구체적 범위가 열거되어 있는가? [명확성, Spec §US3]
- [ ] CHK013 차트 색상이 "다크 모드에서 더 밝은 색상 사용"이라는 기준이 측정 가능한 명암 또는 색상 토큰 정의로 대체되어 있는가? [명확성, 모호성, Spec §US3]
- [ ] CHK014 모바일 화면에서 "필터 항목이 많은 경우"의 임계값(몇 개 이상일 때 스크롤 탭으로 전환하는지)이 명시되어 있는가? [명확성, Gap, Spec §6 엣지 케이스]
- [ ] CHK015 SC-004의 "즉시 반영된다"가 SC-002(200ms) 또는 SC-003(300ms) 수준의 수치 기준으로 정의되어 있는가? [명확성, Spec §SC-004]
- [ ] CHK016 SC-006의 WCAG AA 명암비(4.5:1) 기준이 적용되는 요소 범위(본문 텍스트만인지, 아이콘·차트 레이블 포함인지)가 명시되어 있는가? [명확성, Spec §SC-006]
- [ ] CHK017 "헤더 또는 내비게이션 영역"이라는 테마 토글 위치 표현이 하나의 확정된 위치로 좁혀져 있는가? [명확성, 모호성, Spec §FR-009]

## 요구사항 일관성 (Requirement Consistency)

- [ ] CHK018 §6 엣지 케이스의 "브라우저 세션 내에서만 유지"와 §8 가정 사항의 "localStorage" 저장이라는 상충되는 표현이 하나의 방식으로 통일되어 있는가? [일관성, 충돌, Spec §6, §8]
- [ ] CHK019 OS 다크 모드 설정(FR-012)과 localStorage 저장값(FR-011) 중 어느 것이 초기 테마 결정 시 우선순위를 가지는지 명시되어 있는가? [일관성, Gap, Spec §FR-011, FR-012]
- [ ] CHK020 관리자 페이지에서 지표를 비활성화하면 대시보드의 KPI 카드와 차트 필터 지표 목록 양쪽 모두에 반영되는지, 일부에만 반영되는지 일관된 범위가 명시되어 있는가? [일관성, Gap, Spec §FR-017]
- [ ] CHK021 기간 필터 단위("1주, 1개월, 3개월, 1년")가 사용자 스토리(US2), 기능 요구사항(FR-005), 시나리오(§9) 전반에서 동일하게 사용되고 있는가? [일관성, Spec §US2, FR-005]

## 수락 기준 품질 (Acceptance Criteria Quality)

- [ ] CHK022 "차트가 즉시 업데이트된다"(US2 수락 기준)는 SC-002(200ms)와 명시적으로 연결되거나 동일한 수치로 정의되어 있는가? [수락 기준, Spec §US2, SC-002]
- [ ] CHK023 차트 유형 전환 시 "이전 애니메이션을 취소하고 새 유형으로 전환한다"는 엣지 케이스가 US2 또는 FR-007의 수락 기준에 포함되어 있는가? [수락 기준, Gap, Spec §6 엣지 케이스]
- [ ] CHK024 관리자 페이지에서 마지막 활성 지표를 비활성화 시도 시 "안내 메시지 표시"라는 동작이 FR-017 또는 US4 수락 기준에 반영되어 있는가? [수락 기준, Gap, Spec §6 엣지 케이스]
- [ ] CHK025 "테마 전환 시 CSS 전환(transition)으로 깜빡임 없이 전환"이라는 엣지 케이스가 US3 또는 SC-003의 수락 기준으로 형식화되어 있는가? [수락 기준, Gap, Spec §6 엣지 케이스]

## 시나리오 커버리지 (Scenario Coverage)

- [ ] CHK026 localStorage에 저장된 테마 값이 있는 상태에서 OS 다크 모드가 변경되었을 때의 처리 시나리오가 명시되어 있는가? [시나리오 커버리지, Gap, Spec §US3]
- [ ] CHK027 기간 필터와 지표 필터를 동시에 변경하는 경우의 우선순위 또는 처리 순서 시나리오가 정의되어 있는가? [시나리오 커버리지, Gap, Spec §US2]
- [ ] CHK028 관리자가 지표를 비활성화한 후 해당 지표만 남은 차트에서의 동작(빈 차트 표시, 차트 숨김 등)이 시나리오로 정의되어 있는가? [시나리오 커버리지, Gap, Spec §US4]
- [ ] CHK029 사용자가 모바일 환경에서 차트 필터를 사용하는 주요 흐름 시나리오가 §9 시나리오 섹션에 포함되어 있는가? [시나리오 커버리지, Gap, Spec §9]

## 비기능 요구사항 명확성 (Non-Functional Requirements Clarity)

- [ ] CHK030 키보드 탐색 요구사항(접근성)이 "모든 인터랙티브 요소"로 명시되어 있는데, 차트 필터 컨트롤·테마 토글·관리자 토글 각각에 대해 구체적으로 적용됨이 확인되는가? [비기능, 명확성, Spec §7]
- [ ] CHK031 "화면 블로킹 없이 즉시 업데이트"라는 성능 요구사항이 구체적인 측정 가능 기준(예: 메인 스레드 블로킹 없음, requestAnimationFrame 사용 등)으로 정의되어 있는가? [비기능, 명확성, Spec §7]
- [ ] CHK032 모바일(375px) 해상도에서 차트 가독성(최소 폰트 크기, 차트 높이 등)에 대한 요구사항이 명시되어 있는가? [비기능, Gap, Spec §FR-018]
