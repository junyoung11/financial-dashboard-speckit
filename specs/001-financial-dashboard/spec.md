# 금융 분석 대시보드 데모 기능 명세

**작성일**: 2026-04-29  
**버전**: 1.0.0  
**상태**: 초안

---

## 1. 개요

기존 Next.js/React 기반 프로젝트에 금융 분석 데모 대시보드를 추가한다. 이 기능은 실제 금융 서비스처럼 보이는 데모 화면을 제공하며, 금융 데이터 탐색, 차트 기반 분석, 관리자 화면 구성을 시각적으로 보여주는 것을 목적으로 한다.

### 배경 및 동기

금융 분석 대시보드 데모를 통해 다음 역량을 시각적으로 시연한다:
- 금융 지표 시각화 (KPI 카드, 차트)
- 인터랙티브 차트 필터 (기간, 지표, 차트 유형)
- 라이트/다크 모드 일관성
- 관리자 전용 구성 화면

모든 데이터는 샘플(목업) 데이터로 구성되며, 실제 금융 API 연동은 포함하지 않는다.

### 범위

**포함**:
- 금융 지표 KPI 카드 섹션이 있는 대시보드 메인 화면
- 다양한 차트 유형 표시 (라인, 바, 영역, 도넛)
- 차트별 필터 (기간, 지표, 차트 유형)
- 라이트/다크 모드 전환 기능
- 전체 UI 테마 일관성 (카드, 표, 차트, 버튼, 배지)
- 관리자 전용 페이지 (별도 라우트)
- 관리자 페이지에서 지표 목록, 차트 구성, 샘플 데이터 상태 확인 및 관리

**제외**:
- 실제 금융 API 또는 외부 데이터 소스 연동
- 실제 사용자 인증/로그인 기능 (데모용으로 관리자 페이지는 별도 URL로 접근)
- 실시간 데이터 스트리밍
- 데이터 내보내기(Export) 기능

## 2. 사용자 스토리

### US1. 금융 대시보드 조회 (P1)

**스토리**: 일반 사용자로서, 대시보드 메인 화면에서 주요 금융 지표 카드와 여러 차트를 한 화면에서 확인하여 금융 데이터의 전체 현황을 파악하고 싶다.

**수락 기준**:
- [ ] 대시보드 메인 화면에 최소 4개의 금융 지표 KPI 카드가 표시된다 (예: 포트폴리오 가치, 수익률, 변동성, 거래량)
- [ ] 각 KPI 카드에는 지표명, 현재 값, 전일 대비 변동률(증감 표시)이 표시된다
- [ ] 메인 화면에 최소 3개의 차트가 표시된다 (라인 차트, 바 차트, 도넛/파이 차트 각 1개 이상)
- [ ] 모든 차트는 샘플 금융 데이터로 채워져 있다
- [ ] 화면은 가로 스크롤 없이 데스크탑(1280px 이상)에서 한눈에 볼 수 있다

### US2. 차트 필터 조작 (P1)

**스토리**: 일반 사용자로서, 각 차트의 기간, 지표, 차트 유형을 변경하여 원하는 형태로 데이터를 분석하고 싶다.

**수락 기준**:
- [ ] 각 차트 컴포넌트에 기간 필터가 존재하며, 최소 선택지는 1주, 1개월, 3개월, 1년이다
- [ ] 각 차트 컴포넌트에 지표 필터가 존재하며, 최소 2개 이상의 지표를 전환할 수 있다 (예: 주가 ↔ 수익률, 거래량 ↔ 매수금액)
- [ ] 각 차트 컴포넌트에 차트 유형 필터가 존재하며, 최소 2가지 유형을 전환할 수 있다 (예: 라인 ↔ 바 ↔ 영역)
- [ ] 필터 변경 시 차트가 즉시 업데이트된다 (페이지 새로고침 없이)
- [ ] 여러 차트의 필터가 독립적으로 동작한다 (한 차트의 필터 변경이 다른 차트에 영향을 주지 않는다)

### US3. 라이트/다크 모드 전환 (P1)

**스토리**: 사용자로서, 대시보드 상단에서 라이트 모드와 다크 모드를 직접 전환하여 선호하는 환경에서 사용하고 싶다.

**수락 기준**:
- [ ] 대시보드 헤더 또는 내비게이션 영역에 테마 전환 버튼/토글이 존재한다
- [ ] 테마 전환 시 배경, 텍스트, 카드, 표, 차트, 버튼, 배지 등 모든 UI 요소의 색상이 즉시 전환된다
- [ ] 차트의 색상 팔레트도 테마에 맞게 변경된다 (다크 모드에서 더 밝은 색상 사용)
- [ ] 페이지를 새로 고침해도 마지막으로 선택한 테마가 유지된다 (localStorage 활용)
- [ ] OS의 다크 모드 설정을 초기값으로 반영한다

### US4. 관리자 페이지 접근 및 관리 (P2)

**스토리**: 관리자로서, 별도의 관리자 페이지에서 대시보드에 표시되는 지표 목록, 차트 구성, 샘플 데이터 상태를 확인하고 간단히 관리하고 싶다.

**수락 기준**:
- [ ] 관리자 페이지는 별도의 URL 경로(예: `/admin`)로 접근 가능하며, 일반 대시보드 UI와 시각적으로 구분된다
- [ ] 관리자 페이지에서 현재 대시보드에 표시되는 지표 목록을 확인할 수 있다
- [ ] 관리자 페이지에서 차트 구성(차트 유형, 연결된 지표)을 확인할 수 있다
- [ ] 관리자 페이지에서 각 샘플 데이터 세트의 상태(항목 수, 날짜 범위)를 확인할 수 있다
- [ ] 관리자 페이지에서 지표의 표시 여부를 토글(활성/비활성)할 수 있으며, 변경 사항은 대시보드에 즉시 반영된다 (데모 세션 내에서만 유효)

## 4. 성공 기준

### 측정 가능한 결과

- **SC-001**: 대시보드 메인 화면에서 KPI 카드 4개 이상, 차트 3개 이상이 한 화면에 표시된다
- **SC-002**: 차트 필터 변경 후 200ms 이내에 차트가 업데이트된다
- **SC-003**: 라이트/다크 모드 전환 후 모든 UI 요소가 300ms 이내에 테마에 맞는 색상으로 전환된다
- **SC-004**: 관리자 페이지에서 지표 토글 후 대시보드에서 해당 지표의 표시 여부가 즉시 반영된다
- **SC-005**: 데스크탑(1280px), 태블릿(768px), 모바일(375px) 해상도에서 가로 스크롤 없이 정상 렌더링된다
- **SC-006**: 라이트/다크 모드 각각에서 모든 텍스트의 명암비가 4.5:1 이상을 충족한다 (WCAG AA 기준)

## 6. 엣지 케이스 및 에러 처리

- **빈 데이터 상태**: 선택된 필터 조합에 데이터가 없는 경우, "데이터 없음" 안내 메시지와 빈 차트 영역을 표시한다
- **차트 유형 전환**: 차트 유형을 전환할 때 기존 애니메이션이 완료되지 않은 경우 이전 애니메이션을 취소하고 새 유형으로 전환한다
- **테마 전환 중 레더링**: 테마 전환 시 CSS 전환(transition)을 적용하여 깜빡임 없이 전환한다
- **관리자 지표 비활성화**: 모든 지표가 비활성화된 경우, 최소 1개는 활성 상태를 유지하도록 안내 메시지를 표시한다
- **관리자 페이지 직접 접근**: 데모 환경이므로 별도 인증 없이 접근 가능하나, 일반 대시보드 내비게이션에는 관리자 링크를 노출하지 않는다
- **모바일 화면의 차트 필터**: 필터 항목이 많은 경우 스크롤 가능한 탭 형태로 표시한다

## 7. 비기능 요구사항

- **성능**: 초기 페이지 로딩 후 차트 필터 변경 시 화면 블로킹 없이 즉시 업데이트 (클라이언트 사이드 데이터 처리)
- **접근성**: 모든 인터랙티브 요소는 키보드 탐색 가능, 모든 텍스트는 WCAG AA 명암비 기준 충족
- **반응형**: 모바일(375px), 태블릿(768px), 데스크탑(1280px 이상) 레이아웃 지원
- **테마 일관성**: 라이트/다크 모드 각각에서 Tailwind CSS 디자인 시스템 기반으로 일관된 색상 사용

---

## 8. 가정 사항

- 모든 금융 데이터는 하드코딩된 샘플(목업) 데이터를 사용한다 (외부 API 연동 없음)
- 관리자 페이지의 지표 토글 상태는 브라우저 세션(또는 localStorage) 내에서만 유지된다 (서버 저장 없음)
- 관리자 페이지에 접근 제어(인증)는 구현하지 않는다 (데모 목적)
- 차트 라이브러리로 기존 프로젝트의 Chart.js (react-chartjs-2)를 사용한다
- Tailwind CSS의 `dark` 클래스 전략(class 기반)을 사용한다
- Next.js App Router 구조를 따른다

## 9. 사용자 시나리오 및 테스트

### 주요 흐름 — 대시보드 조회 및 차트 필터 사용

1. 사용자가 대시보드 메인 페이지(`/`)에 접속한다
2. KPI 카드 4개(포트폴리오 가치, 수익률, 변동성, 거래량)가 표시된다
3. 화면 하단에 라인 차트, 바 차트, 도넛 차트가 표시된다
4. 사용자가 라인 차트의 기간 필터를 "1개월"에서 "3개월"로 변경한다
5. 라인 차트의 데이터가 즉시 업데이트된다
6. 사용자가 차트 유형을 "라인"에서 "영역"으로 변경한다
7. 차트 유형이 전환된다
8. 다른 차트들은 변경되지 않는다

### 주요 흐름 — 테마 전환

1. 사용자가 대시보드에 접속한다 (OS 설정에 따라 초기 테마 결정)
2. 헤더의 테마 토글 버튼을 클릭한다
3. 전체 UI가 다크 모드로 전환된다 (배경, 카드, 차트, 버튼 등)
4. 사용자가 페이지를 새로고침한다
5. 다크 모드가 유지된다
6. 테마 버튼을 다시 클릭하면 라이트 모드로 돌아간다

### 주요 흐름 — 관리자 페이지 관리

1. 관리자가 `/admin` URL로 접속한다
2. 관리자 전용 헤더와 레이아웃이 표시된다
3. 지표 관리 표에서 현재 활성화된 지표 목록을 확인한다
4. "거래량" 지표의 토글을 비활성으로 변경한다
5. 대시보드(`/`)로 이동하면 거래량 KPI 카드가 표시되지 않는다

### 예외 흐름 — 데이터 없음

1. 사용자가 차트에서 특정 필터 조합을 선택한다
2. 해당 조합에 데이터가 없는 경우
3. 차트 영역에 "선택한 기간의 데이터가 없습니다" 메시지가 표시된다
4. 다른 차트 및 KPI 카드는 영향받지 않는다
